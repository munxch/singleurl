# Mino Results Pages â€” Action-Oriented Strategy

## Core Principle

**Don't return information. Return recommendations.**

Everyone else: "Here's data."
Mino: "Here's what to do, and why."

The results page isn't a report. It's a decision delivered.

---

## Results Page Structure (All Examples)

Every results page follows this hierarchy:

| Section | Purpose | Priority |
|---------|---------|----------|
| **Recommendation** | What should I do? | Loudest |
| **Rationale** | Why this one? (2-3 bullets) | Supports decision |
| **Primary Action** | One-click to do it | Unmissable |
| **Urgency/Context** | Why now? What if I wait? | Creates momentum |
| **Alternatives** | Other options | Secondary/collapsed |
| **Methodology** | How Mino got here | Trust-building (subtle) |
